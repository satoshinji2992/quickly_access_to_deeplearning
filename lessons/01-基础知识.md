# y = ax + b! 神经网络到底是什么?

什么是神经网络? 神经网络其实就是一个非常复杂的函数, 我们把它叫做 $f$ 吧.

我们现在有一个输入 $x$($x$ 和 $y$ 不一定是单变量, 通常是一个 tensor), 把 $x$ 塞进神经网络, 就能得到输出 $y=f(x)$.

这里的 $x$ 和 $y$ 可以是几乎任何有关联的事物. 比如图像识别任务里, $x$ 是一张图片, $y$ 是图片的内容; ChatGPT 里, $x$ 是你的 prompt 和 question, $y$ 是 ChatGPT 的回答.

那神经网络是怎么完成这些复杂任务的?

一上来可能难以理解, 让我们从简单的任务开始看起来吧.

---

## 任务一: 拟合一条直线

![任务一示意图](../assets/images/mission_0.png)

这是一个坐标系, 上面有一些点, 你要让你的电脑告诉你当 $x=5$ 时, $y$ 是多少.

看着很简单, 你一眼就能看出规律大概是一条直线. 但电脑要怎么知道呢?

你想到了, 既然这些点分布在一条线上, 那描述这条线的函数一定长这样:

$$y = ax + b$$

这便是你的模型. 只要能找到最合适的 $a$ 和 $b$, 问题就解决了.

于是第一关要做的事情很朴素:

- 用 `numpy` 和 `pandas` 读入数据.
- 写出线性模型 $y=ax+b$.
- 用 MSE 衡量预测和真实值的差距.
- 用梯度下降不断调整 $a$ 和 $b$.

做到这里, 你就已经碰到了深度学习里最核心的几个词: 模型、参数、损失函数、梯度、学习率、训练.

## 任务二: 使用神经网络预测点在圆形内还是圆形外

第一个任务里, 你让电脑学会了画一条最合适的线.

但是现在的任务可没那么容易了.

给定一个平面上的点 $(x,y)$, 让神经网络告诉我们这个点是在圆内还是圆外.

![圆形分类数据](../assets/images/data_circle.png)

如果还用 $ax+by+c$ 这种线性模型, 它最多只能画出一条直线. 圆的边界是曲线, 所以它画不出来.

这时你就不得不引入新的东西:

- 激活函数, 比如 ReLU, 让模型不再只是直线.
- Softmax, 把 logits 变成概率.
- 交叉熵损失, 衡量分类预测到底有多离谱.
- batch 和 epoch, 让模型可以用一批一批的数据训练.
- 反向传播, 把最后的损失一步步传回每一层参数.

这一关的重点不是“套一个神经网络”, 而是亲手看见一件事: 多层线性变换加上非线性激活, 可以把一条直线变成弯曲的分类边界.

详细任务在 [task_01_circle_classifier](../exercises/block_01_basics/task_01_circle_classifier/README.md).

## 任务三: 完善你的深度学习库

做完圆形分类以后, 你大概率已经有了一个能跑的 MLP.

但代码也可能已经开始有点难看了.

参数初始化、前向传播、ReLU、Softmax、交叉熵、反向传播、参数更新, 全挤在一个类里. 这不是不能用, 只是后面很难改.

比如你想把 ReLU 换成 GELU, 把 SGD 换成 Momentum, 多加一层隐藏层, 或者把圆形分类换成 MNIST. 如果每次都要钻进一大坨代码里改 forward 和 backward, 很快就会崩溃.

所以第三关不是换一个更难的数据集, 而是先收拾工具.

你要把网络拆成几个更清楚的模块:

- Layer: 负责一层的 forward 和 backward.
- Loss: 负责衡量预测和标签的差距.
- Optimizer: 负责根据梯度更新参数.
- Sequential: 负责把层串起来.

旧稿里还会顺带讲到激活函数、优化器、归一化、正则化和初始化. 这些东西不是为了背名词, 而是为了回答训练时经常遇到的问题: loss 为什么不动? 为什么发散? 为什么训练集很好测试集很差? 为什么 batch 一小就不稳定?

详细任务在 [task_02_mini_dl_lib](../exercises/block_01_basics/task_02_mini_dl_lib/README.md).

## 任务四: 用 MLP 识别 MNIST

到这里, 你已经会处理二维点了.

但真实图片不是两个数字. 一张 MNIST 手写数字图片是 $28\times 28$ 个像素, 展平成向量以后就是 784 个输入.

我们先不急着上卷积神经网络, 先让 MLP 硬吃这个任务:

$$28 \times 28 \rightarrow 784 \rightarrow 128 \rightarrow 10$$

它当然能学到一些东西. 但你也会发现, 把图片拉平成一串数字以后, 图片的空间结构就没了. 左边和右边、上面和下面、笔画之间的局部关系, MLP 都要自己从 784 个数字里慢慢猜.

这就自然引出了下一章: CNN 和 ResNet.

详细任务在 [task_03_mnist_mlp](../exercises/block_01_basics/task_03_mnist_mlp/README.md).

## 做完这一章以后

做完 Block 1, 你应该不只是“跑通了几个脚本”.

你应该能看懂一个训练循环为什么长这样:

```text
forward -> loss -> backward -> update
```

也应该能知道, 深度学习里的很多词并不神秘:

- 参数就是要被训练出来的数字.
- loss 是模型现在有多差的度量.
- 梯度告诉你参数应该往哪里改.
- 激活函数让模型能表达非线性.
- batch 让训练在速度和稳定之间折中.
- optimizer 决定参数怎么走.

下一章我们会把这个过程搬到图像上, 用 ResNet 去分类真实图片.
