# Block 1：基础知识

这一块先不急着谈 ResNet、Transformer、大模型。那些东西当然很酷，但如果你还没亲手把一个 loss 降下来，它们只会像一堆漂亮但抓不住的名词。

我们先从很小的东西开始。

你会先让电脑学会一条线。这个任务小到有点朴素：只有两个参数 `a` 和 `b`。但别嫌它简单，神经网络训练的核心闭环已经全在里面了：

```text
给输入 -> 算预测 -> 算 loss -> 算梯度 -> 改参数 -> 再来一遍
```

后面所有模型都在重复这件事。区别只是参数从 2 个变成几千个、几百万个、几十亿个；数据从一列数字变成图片、文本、语音；函数从一条直线变成很多层矩阵和非线性函数的组合。

## 这一块你会经历什么

第一关是线性回归。

你看到一堆点，觉得它们差不多在一条线上。人眼一眼能看出来的东西，电脑看不出来。电脑只能拿着 `a` 和 `b` 去试，试完以后算自己错了多少，再一点点调整。这个过程会引出模型、参数、MSE、梯度下降和学习率。

第二关是圆形分类。

上一关一条线够用了，这一关不够了。你要判断点在圆内还是圆外，边界是弯的。线性模型怎么调都只能画直线，于是你会自然遇到“模型表达能力不够”的问题。为了解决它，我们引入 ReLU、隐藏层、MLP、Softmax、交叉熵和反向传播。

第三关是小型深度学习库。

模型跑起来之后，你会发现代码开始变乱。激活函数写在模型里，优化器写在 `backward` 里，loss 和训练循环互相纠缠。你当然可以继续复制粘贴，但这不是研究者的工作方式。我们要把 Layer、Loss、Optimizer 拆出来，让代码变成可以继续长大的东西。

第四关是 MNIST。

这时你已经有了一个小库。我们拿它去处理真实图片：手写数字。做法很粗暴，把 `28 x 28` 的图片拉平成 784 维向量，再丢给 MLP。它能学，但你会明显感觉哪里不太对。图片的局部结构被打散了。这个“不太对”，就是下一块 ResNet 的入口。

## 怎么判断自己学会了

不要用“代码能跑”作为唯一标准。代码能跑只是第一步。

完成 Block 1 后，你应该能用自己的话回答：

- 为什么模型需要参数？
- loss 到底在衡量什么？
- 梯度为什么能告诉参数往哪边走？
- 学习率为什么会让训练变慢、变快、甚至直接炸掉？
- 为什么线性模型分不开圆？
- ReLU 为什么能让模型画出弯曲边界？
- Softmax 输出的概率和 logits 有什么区别？
- 交叉熵为什么适合分类？
- `forward` 和 `backward` 分别在算什么？
- 为什么要把模型拆成 Layer、Loss、Optimizer？
- 为什么 MLP 处理图片时会显得别扭？

如果这些问题你只是“好像懂”，那就回到实验里改参数。深度学习很多东西不是读懂的，是你亲手把 loss 调坏又调回来之后才真的懂。
