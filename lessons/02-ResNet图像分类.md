# Block 2：ResNet 图像分类

上一块最后，我们用 MLP 处理了 MNIST。它能跑，也能学到一些东西，但你应该会觉得有点别扭。

一张图片不是一串普通数字。像素之间有位置关系，有局部结构，有边缘、纹理、形状。把图片直接拉平成向量，相当于把这些关系都拆散，然后指望模型自己重新摸出来。它不是完全做不到，但太浪费。

这一块我们进入图像模型。目标不是直接调用 PyTorch 写一个 ResNet，而是沿着自己的 NumPy 小库继续往前走：先让图片正确进入训练循环，再手写卷积、池化、BatchNorm、残差块，最后组装一个轻量 ResNet 去训练 CIFAR-100。

这会比 Block 1 硬很多。尤其是 `Conv2D.backward` 和 `BatchNorm2D.backward`，第一次写很可能会错。没关系。这一块真正要训练的是研究者习惯：写 shape、做梯度检查、先小样本过拟合，再完整训练。

## 为什么不能继续只用 MLP

MNIST 的数字很干净，背景简单，图片也小。你把 `28 x 28` 拉平成 784 维向量，MLP 还能学。

CIFAR-100 就不一样了。它有 100 个类别，每张图只有 `32 x 32`，主体可能很小，背景可能很乱，同一个类别内部差异也很大。

如果继续 flatten，模型会遇到几个问题：

- 相邻像素的空间关系被拆散了。
- 左上角的一条边和右下角的一条边会被当成完全不同的输入位置。
- 参数量容易变大，但没有利用图片的局部性。
- 模型需要自己从头学“平移后的同一个特征仍然相似”。

卷积的出现，就是为了把这些图片先验放进模型结构里。

一个 `3 x 3` 卷积核只看局部窗口，而且同一个卷积核会在整张图上共享使用。这样它学到的“边缘检测器”不只在某个固定位置有效，而是在很多位置都能用。

这不是偷懒，而是把我们对图片的认识写进模型。

## 这一块你会经历什么

第一步是数据管线。图片数据有通道、高度、宽度。你要知道 `NHWC` 和 `NCHW` 的区别，知道为什么要标准化，为什么训练时要随机裁剪和翻转。

第二步是卷积。卷积不是魔法，它就是一个小窗口在图片上滑动。为了算得快，我们用 `im2col` 把很多局部窗口展开成矩阵，再用矩阵乘法完成卷积。

第三步是 CNN 常用层。池化负责降采样，GlobalAvgPool 负责把空间维汇聚掉，BatchNorm 负责让训练更稳。

第四步是残差块。网络变深后，不一定更好训练。ResNet 的 shortcut 让输入可以绕过几层直接往后走，也让梯度多一条路回去。

第五步是训练。你会先做 500 张图片的小样本过拟合测试。如果小样本都学不住，不要急着跑完整 CIFAR-100，先回去查算子和训练循环。

最后一步是报告。模型训练完不等于你理解了它。你要用对照实验回答：数据增强有没有用？BN 有没有用？残差连接有没有用？

## 每个任务的产出

task_10 结束时，你应该有一个可靠的数据入口。给它一组图片和标签，它能输出模型需要的 batch；打开随机增强时结果有变化，固定 seed 时结果能复现。

task_11 结束时，你应该能解释一层卷积从输入到输出的每个 shape。你不只是知道“卷积会提特征”，还知道 `dW`、`db`、`dx` 是怎么从矩阵乘法里来的。

task_12 结束时，你应该能把 CNN 训练中常见的三类层讲清楚：池化为什么降采样，GlobalAvgPool 为什么适合分类，BatchNorm 为什么训练和验证行为不同。

task_13 结束时，你应该写出一个残差块，并理解 shortcut 不是装饰品。它同时影响前向的信息保留和反向的梯度传播。

task_14 结束时，你应该有一个能训练的轻量 ResNet 项目。哪怕准确率不高，也要能完成小样本过拟合、完整训练、日志记录和 checkpoint 恢复。

task_15 结束时，你应该能写出一份有证据的实验报告。报告里不是“我觉得 BN 有用”，而是“在这个设置下，去掉 BN 后 loss 怎么变、acc 怎么变”。

## 常见卡点

第一个卡点是 shape。

图像模型里，很多错误不是数学公式错，而是维度顺序错。`NCHW` 和 `NHWC` 一旦混了，模型可能直接报错，也可能更糟：能跑，但学不到。

第二个卡点是反向传播。

`Conv2D.backward`、`MaxPool2D.backward`、`BatchNorm2D.backward` 都比 Block 1 的线性层复杂。不要靠眼睛猜对。小输入、数值梯度检查、逐层打印梯度范数，是这块最重要的调试工具。

第三个卡点是训练系统。

模型单层测试都对，不代表训练一定对。学习率、数据增强、BatchNorm 模式、标签 shuffle、checkpoint 保存，都可能让结果变怪。

第四个卡点是实验解释。

如果你只跑一个结果，很难知道它为什么好或为什么差。ResNet 这一块从一开始就要养成做对照实验的习惯。

## 这一块结束时你应该会什么

你应该能解释：

- 为什么图片任务需要卷积，而不是只靠 MLP。
- `NCHW` 里每个维度是什么意思。
- 标准化、随机裁剪、水平翻转分别在解决什么。
- `im2col` 为什么能把卷积变成矩阵乘法。
- `Conv2D.backward` 里 `dW`、`db`、`dx` 分别从哪里来。
- MaxPool 反向传播为什么只把梯度传给最大值位置。
- BatchNorm 训练和推理为什么不一样。
- 残差连接为什么能帮助深层网络训练。
- 小样本过拟合测试为什么是调试模型的好办法。
- 消融实验为什么要一次只改一个变量。

如果你能把一个错误的 ResNet 训练过程拆开定位到“数据、算子、结构、优化”中的某一类问题，这一块就真的有收获了。
