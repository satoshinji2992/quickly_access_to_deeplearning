# ResNet 图像分类

## 为什么这里不能再用 MLP 硬上

如果把一张 `32×32×3` 的图片直接拉平成一个向量，再接全连接层，参数会迅速膨胀，而且最关键的是：图像的空间结构会被打散。

卷积层的好处就在于，它会用同一组小卷积核在整张图上重复扫描，天然更适合找边缘、纹理和局部部件。

![CNN 和 MLP 的区别](../assets/images/cnn_vs_mlp.svg)

## ResNet 到底解决了什么

卷积网络能处理图像，但网络一旦变深，就会出现退化问题：不是测试集差，而是训练集本身都很难学好。

ResNet 的关键想法是：

$$H(x) = F(x) + x$$

也就是说，不强迫每一层都从头学一个完整映射，而是让它学“在原输入基础上该改多少”。

## 最基础的残差块是什么样

一个最常见的 basic block，大致就是：

1. 卷积
2. BN
3. ReLU
4. 再来一层卷积
5. 再做一次 BN
6. 加上 shortcut
7. 再过 ReLU

![残差块示意图](../assets/images/residual_block_flow.svg)

这里最重要的直觉是：shortcut 让梯度和信息都多了一条近路，不容易在深层网络里越传越弱。

## 图像实验完整流程长什么样

这一章真正想带你做的是一个完整实验，而不只是把模型搭出来。

最基础的流程是：

1. 配好 CIFAR-100 和数据增强
2. 写出最小的 `BasicBlock`
3. 组装简化版 `ResNet`
4. 写训练循环和验证循环
5. 观察 loss 和 accuracy
6. 记录实验

![ResNet 训练流程](../assets/images/resnet_training_pipeline.svg)

## 这章最应该盯住什么

做图像分类时，不要只盯 `val_acc` 一个数字。至少同时看：

- train loss
- train acc
- val loss
- val acc

这样你才能区分：

- 是不是没学起来
- 是不是过拟合了
- 是不是学习率不对
- 是不是数据预处理有问题

## 这一章的目标

你不需要一开始就复刻论文里的完整 ResNet。更好的路线是：

- 先写最小残差块
- 再组一个简化版 ResNet
- 再跑 CIFAR-100
- 最后才慢慢加调度器、保存权重、画曲线这些工程细节
