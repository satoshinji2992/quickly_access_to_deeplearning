# RL

强化学习在大模型里经常和“对齐”一起出现.

它不是预训练的主菜, 更像是在模型已经会说话以后, 用偏好信号继续调整行为.

## PPO

PPO 是 RLHF 里常见的优化方法.

大致流程:

```text
模型生成回答
reward model 打分
PPO 根据奖励更新模型
```

PPO 会限制模型不要一次偏离原模型太远, 否则语言质量容易崩.

## GRPO

GRPO 是近年常见的 PPO 变体思路之一.

它会用一组回答之间的相对表现来估计优势, 减少对 value model 的依赖.

在推理、数学、代码类任务中, 这类方法经常和可验证奖励结合.

## 先记住什么?

强化学习不是让模型从零学语言.

它通常建立在预训练和 SFT 之后, 用奖励信号继续塑形.
