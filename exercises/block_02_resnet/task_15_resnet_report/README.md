# task_15：ResNet 实验报告

训练模型很容易让人上头。loss 降了，accuracy 涨了，好像就赢了。

但如果你想成为研究者，只会跑出一个数字是不够的。你要能解释这个数字从哪里来，哪些设计真的帮了忙，哪些只是看起来高级，哪些结论其实还没有证据。

这一节要完成 [REPORT.md](/root/code/python/quickly_access_to_deeplearning/exercises/block_02_resnet/task_15_resnet_report/REPORT.md)。它不是形式作业，而是把前面所有代码变成“实验结论”的过程。

## 1. 这个任务解决什么问题

假设你训练出了一个 ResNet，验证准确率是 28%。

这个数字本身能说明什么？

其实很有限。

我们还想知道：

- 如果不用数据增强，会怎样？
- 如果去掉 BatchNorm，会怎样？
- 如果去掉残差连接，只保留普通 CNN，会怎样？
- 模型最常分错哪些类别？
- 训练集很好、验证集不好，是过拟合还是数据增强不足？
- loss 降了但 accuracy 不涨，说明什么？

这些问题都不能靠单次训练回答。你需要设计对照实验。

对照实验的意思是：一次只改一个因素，其他设置尽量保持一致。这样你才能比较清楚地说，指标变化更可能来自这个因素。

## 2. 如何让模型学会这个？

严格说，实验报告不是让模型学习，而是让你学习。

模型训练时，只会根据梯度更新参数；它不会自动告诉你“我为什么变好了”。研究者要做的事情，是从一组组实验里读出模式。

比如你比较两组实验：

```text
A: 无数据增强
B: 随机裁剪 + 翻转
```

你可能看到：

```text
A: train acc 高，val acc 低
B: train acc 略低，val acc 更高
```

这时候可以得出一个谨慎的结论：数据增强让训练更难了一点，但提高了泛化能力。

注意“谨慎”。不要写成：

```text
数据增强彻底解决了过拟合问题。
```

除非你的实验真的证明了“彻底解决”。大多数时候，我们只能说“在本实验设置下，观察到某某现象”。

再比如 BatchNorm：

```text
无 BN: loss 抖动，收敛慢
有 BN: loss 更平滑，训练更快
```

这个结论就比单纯说“BN 有用”更像研究报告。因为它说明了有用在哪里。

## 3. 需要引入哪些数学对象

实验报告里最重要的数学对象不是复杂公式，而是指标和比较。

第一，训练损失：

```text
train loss
```

它告诉你模型在训练数据上是否学到了东西。训练损失不降，通常说明优化、梯度、学习率或实现有问题。

第二，训练准确率：

```text
train acc
```

它告诉你模型对训练集的分类能力。训练准确率很高但验证准确率低，往往是过拟合。

第三，验证损失和验证准确率：

```text
val loss
val acc
```

它们更接近模型对新数据的表现。注意，有时 val loss 变差但 val acc 还行，说明模型置信度可能有问题。

第四，对照变量。

每组实验都要写清楚你改了什么：

```text
数据增强：关闭 -> 开启
BatchNorm：移除 -> 保留
残差连接：plain CNN -> ResNet
```

第五，随机种子。

深度学习实验有随机性。至少记录：

```text
seed
初始化方式
数据 shuffle
增强策略
```

如果时间允许，同一配置跑多个 seed 会更可靠。但作为课程任务，至少要把单次实验的 seed 写下来。

## 5. 逐步实现顺序

第一步，补全实验设置。

先写清楚：

```text
数据集
训练/验证划分
模型结构
优化器
学习率
batch size
epoch
随机种子
硬件或运行环境
```

这些看起来很琐碎，但没有它们，别人无法复现你的结果。

第二步，跑一个 baseline。

baseline 不需要最强，但要稳定。比如：

```text
轻量 ResNet
基础数据预处理
固定学习率
固定 seed
```

第三步，做数据增强对照。

只改变增强策略：

```text
无增强
随机裁剪 + 水平翻转
```

其他设置保持一样。

第四步，做 BatchNorm 对照。

只改变 BN：

```text
无 BN
有 BN
```

如果去掉 BN 后训练明显不稳定，要记录 loss 曲线，而不是只写最终准确率。

第五步，做残差连接对照。

比较：

```text
plain CNN
ResNet
```

尽量让两者参数规模接近，不然很难判断提升来自残差连接还是参数更多。

第六步，做错误样例分析。

挑几张分错的图，写下：

- 真实类别是什么？
- 模型预测成什么？
- 这两个类别是否视觉上相近？
- 图片是否模糊、遮挡、主体很小？
- 这个错误暗示模型缺什么？

第七步，写总结。

总结不要写空话。好的总结应该像这样：

```text
在本实验设置下，数据增强让 train acc 从 72% 降到 65%，但 val acc 从 24% 提升到 29%，说明增强降低了记忆训练集的能力，但改善了泛化。BatchNorm 主要改善了训练稳定性，去掉后 loss 在前 5 个 epoch 波动明显。残差连接在相近深度下带来更高的训练准确率，说明 plain CNN 的优化更困难。
```

数字可以换成你自己的，但结论要跟数据贴在一起。

## 6. 常见错误和调试方法

最常见的问题是一次改太多东西。

比如你同时改了数据增强、学习率、模型深度和 batch size，然后发现 val acc 提升了。这个提升到底来自谁？说不清。

第二个问题是只报告最好的一次。

如果你跑了很多次，只挑最高分写，报告会偏。至少要说明你挑选的是 best checkpoint，还是最后一个 epoch。

第三个问题是结论超过证据。

看到某组实验高 1%，不能立刻说“显著提升”。课程任务里可以写“略有提升”或“在本次运行中更高”，这更诚实。

第四个问题是不记录失败实验。

失败实验很有价值。比如学习率太大导致 NaN，可以说明训练对学习率敏感；去掉 BN 后不收敛，可以说明 BN 在这个实现里承担了稳定作用。

第五个问题是 train/val 指标混着说。

训练准确率高不代表模型泛化好。报告里要明确区分 train acc 和 val acc。

第六个问题是错误样例分析写得太空。

不要只写“模型分错了”。要观察图片本身：主体小不小，颜色像不像，类别之间有没有细粒度差异。

## 7. 必做实验

至少完成下面三组对照。

1. 数据增强对照。

   比较无增强和随机裁剪 + 水平翻转。记录 train loss、train acc、val loss、val acc。

2. BatchNorm 对照。

   比较有 BN 和无 BN。重点观察训练稳定性、收敛速度和最终验证准确率。

3. 残差连接对照。

   比较 plain CNN 和 ResNet。尽量保持深度或参数规模相近。

4. 错误样例分析。

   至少记录 5 张分错图片。每张写真实标签、预测标签和可能原因。

5. 小样本过拟合记录。

   把 task_14 的 500 张过拟合结果也放进报告。它是证明训练代码可信的重要证据。

如果时间不够，可以缩短 epoch，但不要省掉对照逻辑。

## 8. 完成后应该能回答的问题

完成这一节后，你应该能回答：

- 为什么单个 accuracy 数字不够说明问题？
- 什么叫“一次只改一个变量”？
- 数据增强通常会怎样影响 train acc 和 val acc？
- BatchNorm 的作用应该从哪些现象里观察？
- 残差连接的实验应该怎么设计才公平？
- 为什么要报告 seed 和训练配置？
- best checkpoint 和 final checkpoint 有什么区别？
- 错误样例分析能暴露哪些模型局限？
- 什么时候可以说“提升明显”，什么时候只能说“本次实验中更高”？

这些问题答清楚，你的报告就不是流水账，而是一个小型研究记录。
