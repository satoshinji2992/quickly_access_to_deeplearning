# task_03：用 MLP 识别 MNIST 手写数字

## 1. 这个任务解决什么问题

前面我们一直在二维平面上玩点。现在换一个真实一点的数据：手写数字。

MNIST 每张图片是 `28 x 28` 的灰度图，标签是 0 到 9。你要用上一关的小型深度学习库训练一个 MLP，让它识别这些数字。

这一步的目的不是证明 MLP 是图片分类的最佳方案。恰恰相反，我们要通过它发现：MLP 虽然能处理图片，但它处理得很笨。

## 2. 如何让模型学会这个？

人看数字时，会看笔画。比如 8 有两个圈，1 是一竖，7 有一横加一斜。我们自然会利用局部结构：某个小区域的像素组合成笔画，笔画再组合成数字。

MLP 不知道这些。

我们把图片拉平成向量：

```text
28 x 28 -> 784
```

在 MLP 眼里，这就是 784 个普通数字。第 100 个像素和第 101 个像素是不是相邻，它并没有天然概念。

这很浪费，但也很适合作为过渡任务。因为你会看到：同一套训练逻辑，从二维点换成图片以后仍然能用。

## 3. 需要引入哪些数学对象

原始图片：

$$
X\in \mathbb{R}^{N\times 28\times 28}
$$

拉平后：

$$
X_{\text{flat}}\in \mathbb{R}^{N\times 784}
$$

标签有 10 类：

$$
y\in \mathbb{R}^{N\times 10}
$$

模型可以很简单：

$$
H=\mathrm{ReLU}(XW_1+b_1)
$$

$$
logits=HW_2+b_2
$$

loss 还是交叉熵。

你会发现，任务二里的二分类 MLP 和这里的十分类 MLP，本质差别不大：

- 输入维度从 2 变成 784。
- 输出维度从 2 变成 10。
- 数据从二维点变成图片。

训练闭环没变。

## 5. 逐步实现顺序

第一步，加载 MNIST。

第二步，把图片转成 NumPy 数组，并除以 255：

```python
images = images / 255.0
```

这一步很重要。否则像素范围是 0 到 255，输入尺度太大，训练会更不稳定。

第三步，reshape：

```python
images = images.reshape(-1, 28 * 28)
```

第四步，把标签转成 one-hot：

```python
y[np.arange(labels.size), labels] = 1
```

第五步，定义 MLP。

第六步，训练并打印 loss 和 accuracy。

旧参考代码 `codes/reference/neuralNetwork_test/cnn_test.py` 虽然文件名叫 `cnn_test.py`，但它实际上就是把 MNIST 拉平成向量，再用全连接网络训练。你可以把它当成历史参考。

## 6. 常见错误和调试方法

下载失败：先确认网络。或者换成本地 MNIST 文件加载。

shape 错：`Linear(784, 128)` 只接受 `(N, 784)`。如果你传 `(N, 28, 28)`，矩阵乘法会报错。

准确率一直 10%：这差不多是随机猜。先检查 one-hot，再检查 `argmax(axis=1)`。

loss 不降：检查像素是否除以 255，学习率是否太大。

训练太慢：先只用 5000 张训练，不要一上来跑完整数据集。
